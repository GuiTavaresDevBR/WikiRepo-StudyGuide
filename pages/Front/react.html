React JS

Aqui é onde começa a ter contato com o FRONTEND do mundo real. É importante ressaltar que para inciar os estudos em um framework, é importante que tenha sólidos estudos em Javascript puro, para que entenda o que está acontecendo e realmente saiba trabalhar com React. Possuindo esses conceitos claros, o entendimento e trabalhar com React fica muito mais fácil.

React é uma biblioteca JS para construir interfaces de usuário (UI) de forma mais rápida e eficaz. Ela permite criar interface dividindo a tela em componentes reutilizaveis.React é basicamente focado em:

  • Performance
  • Organização
  • Reaproveitamento de Código
  • Atualização eficiente de tela

Componentes: Componentes são blocos independentes de interface Tudo em react é um componente. Componentes dentro do react sempre irá retornar um JSX.

JSX: É basicamente JavaScript + HTML. É uma sintaxe que parece HTML dentro do Javascript. Aqui conseguimos misturar Tags e Scripts dentro do mesmo "ambiente". Claro que mais para frente iremos aprender as melhores boas práticas para isso. É de extrema importancia saber pequenas regras:

  • Só pode retornar um único elemtno pai por vez.
  • Diferente do HTML, aqui usamos className ao invés de class.
  • Use {} para inserir JavaScript.

Props: São propriedades passadas para componentes. Servem para comunicação entre componentes. Ex: Uma página pode passar dados para outra página, e assim ela receber as informações na mesma.

Renderização: React atualiza a tela automaticamente assim que algum state usa. Isso acontece via Virtual DOM. O React basicamente compara o state antigo com o state novo, e assim atualiza penas o necessário, melhorando a performance nao sendo necessário atualizar a página inteira. Esse conceito é chamado de SPA (Single page Application)

States State = Dados internos do componente. Através de STATES, podemos permitir que a interface se atualize dinamicamente. Aqui iremos aprender o primeiro HOOK de react, chamado de useState.

  • const [numero, setNumero] = useState(0)
  • Nunca tualizamos a variavel numero direto, sempre será através da funcao setNumero.

Eventos Eventos funcionam parecido com o JS normal, mas em camelCase. como por exemplo:

  • onChange
  • onCLick

useEffect Aqui iremos aprender nosso segundo hook do react, qeu é um hook para efeitos coletarais. Ele é disparado sempre que um componente for renderizado , ou quando algum state que voce informar for alterado. Extremamente util para requisicoes, timers, manipulacao externa, APIS, etc.

  • useEffect(() => {fazer algo qd ativado}, [states a serem monitorados])

Condicional JSX Dentro de cada componente, podemos renderizar determinados elementos de acordo com uma condicao. Por exemplo:

  • {logado ? Home : Login }
  • {logado && Dashboard}
